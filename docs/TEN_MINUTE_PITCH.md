## 🎙️ 逐段演說逐字稿與核心論點 (Speech Script)

### 一、 知識圖譜的學術本質與痛點 (00:00 - 01:30)

> **口述重點**：直接從「神經網路的連續語意空間」與「圖譜的離散符號空間」的互補性切入，說明為什麼在 LLM 時代，我們依然需要知識圖譜，以及現有 RAG 的瓶頸。

*   **演說逐字稿參考**：
    > 「各位教授好，今天我要報告的專案是《神經符號 GraphRAG 系統》。在進入系統架構前，我想先精準定義本專案對**知識圖譜 (Knowledge Graph, KG)** 的定位。
    > 
    > 傳統的大語言模型將知識編碼在**連續的高維向量空間**中。這種表徵方式雖然具備極佳的泛化能力，但其致命缺陷在於缺乏邏輯確定性、容易產生幻覺，且無法進行精確的多跳推理（Multi-hop Reasoning）。相較之下，知識圖譜是**離散符號空間**的代表，它透過本體論（Ontology）約束，將知識以結構化的實體與語意關係（SVO 三元組）進行確定性表徵。
    > 
    > 然而，目前業界在結合兩者時（如傳統 RAG 或簡單的 Graph-RAG），面臨兩個核心痛點：
    > 1.  **關係抽象化導致情境丟失 (Context-free)**：知識圖譜的三元組（如 `[李白] -[出生於]-> [碎葉城]`）雖然精確，但丟失了原始文本中的副詞、時序、數量等細節，導致 LLM 無法進行飽滿的問答。
    > 2.  **單點圖譜的擴展性瓶頸**：面對海量跨領域知識時，單一扁平的圖譜會面臨檢索效能下降與實體衝突問題。
    > 
    > 為了克服這些瓶頸，本專案提出了一個**神經符號 (Neuro-Symbolic) 架構下的聯邦分片 GraphRAG 系統**。」

---

### 二、 專案核心設計與創新機制 (01:30 - 06:30)

> **口述重點**：重點介紹「雙層檢索」、「圖外路由 (Graph MoE)」、「圖內回溯 (Symbolic-to-Physical Backtracking)」與「T-Box / A-Box 雙層本體協同」。

*   **演說逐字稿參考**：
    > 「本系統的架構設計在學術上可以拆解為兩個層次：**圖譜外門控路由**與**圖譜內混合檢索**。
    > 
    > **第一，在圖譜外查找層面，我們實作了『混合專家圖譜路由 (Mixture of KGs / Graph MoE)』**：
    > 我們將每一個獨立的領域知識圖譜視為一個獨立的『專家節點 (Expert)』。當用戶輸入查詢時，系統會先提取 Query 中的概念特徵 $Q$。每個專家圖譜則將其主題 Schema 投影為 Key $K$。我們設計了**多屬性門控注意力機制 (Multi-Attribute Gating Attention)**，不僅計算語意相似度，還對齊了 `interest` 與 `professional` 雙維度屬性，計算出門控權重 $\alpha$，動態激活 Top-K 個專家圖譜進行並行聯邦檢索。這避免了單一巨大圖譜的查詢瓶頸。
    > 
    > **第二，在圖譜內查找層面，我們設計了創新的『符號-物理源頭回溯機制 (Symbolic-to-Physical Source Backtracking)』**：
    > 當特定的領域圖譜被激活後，系統執行雙軌檢索。
    > *   **第一軌（符號遍歷）**：在 Neo4j 中執行 1-2 跳的 BFS 拓撲遍歷，獲取精確的 SVO 事實，並保留其物理坐標（即產生此事實的 `chunk_id` 與 `source_doc_id`）。
    > *   **第二軌（源頭回溯）**：這是本專案的核心創新。如果僅靠 SVO 事實不足以答覆，系統會**沿著物理坐標直接回溯至 ChunkStore，拉取原始文件段落（Chunk）的原文細節**。
    > 同時，我們採用**圖譜引導的重排演算法 (Graph-Driven Reranking)**，將圖譜命中的實體數作為偏置項加入向量相似度計算中，優先提取與圖譜事實高度相關的原文。這完美橋接了『離散邏輯邊』與『連續原文細節』。
    > 
    > **第三，在知識表徵上，我們採用了 T-Box 與 A-Box 的雙層協同推理**：
    > 我們預定義了 13 種實體類別與 30 種強類型關係作為 **T-Box（術語域）**的本體約束，防止 OpenIE 關係發散；並將實例文檔抽取的三元組落實在 **A-Box（實例域）**。這種設計確保了系統具備嚴格的語意可歸納性與推理能力。」

### 三、 理論背書與學術基準對標 (06:30 - 08:30)

> **口述重點**：將系統的每個模組映射到頂級學術論文，並對標當前 SOTA GitHub 專案，向教授證明系統的學術完整性與先進性。

*   **演說逐字稿參考**：
    > 「我們的設計並非憑空捏造，而是有著扎實的學術文獻背書：
    > 1.  我們的**圖譜外門控路由**對應了 Shazeer 等人於 2017 年提出的 **Sparsely-Gated MoE** 架構。在 GraphRAG 領域，則對標了 Zhao 等人 (2022) 提出的 **Mixture of Knowledge Graphs (MoKG)** 理論，證實了分治圖推理的擴展性。
    > 2.  我們的**符號-物理回溯機制**呼應了微軟 Edge 等人於 2024 年發表的 **GraphRAG 奠基性工作**，並結合了 ACL 2021 中提出的 **Provenance-Aware Retrieval (溯源感知檢索)** 理論，證明了物理坐標保留對解釋性與抑制幻覺的決定性作用。
    > 3.  **變長度與動態演化**則承襲了 Facebook AI Research (Weston et al., 2014) 的 **Memory Networks** 非參數化記憶思想，利用置換不變性與變長度相容的 Attention 機制，使系統天然兼容動態增減的描述節點。
    > 4.  **聯邦查詢與對齊**則參考了 WWW 2011 的 **FedX** 聯邦優化技術，我們在 `query_shards_parallel` 中實作了分散式圖譜並行檢索。
    > 
    > 在 **SOTA 對標**上，我們參考並優化了目前 GitHub 上的前沿專案：
    > *   相較於浙江大學的 **WKM (World Knowledge Model)**，我們實作了更靈活的 `claude-desktop` Agent 外部對接；
    > *   相較於 King's Lab 的 **ProVe** 三元組校驗，我們在寫入端引入了防幻覺過濾器；
    > *   相較於一般的 **Wikipedia-KG-RAG**，本系統進一步支持跨本地與遠端 Registry 的**聯邦分片並行檢索**，更符合企業級分散式部署需求。」

---

### 四、 專案價值、評估與未來展望 (08:30 - 10:00)

> **口述重點**：說明你如何驗證這個系統（RAGAS 框架、消融實驗），以及系統的未來延展性，展現科學的研究方法。

*   **演說逐字稿參考**：
    > 「為了解決學術界對 RAG 系統的實證質疑，我們引入了 **RAGAS 評估框架 (Retrieval Augmented Generation Assessment)**，從三個維度進行量化評估：
    > *   **Faithfulness（忠實度）**：驗證防幻覺過濾器與自我精煉機制對幻覺的抑制。
    > *   **Answer Relevance（答案相關性）**：評估概念提取的精準度。
    > *   **Context Recall（檢索召回率）**：驗證圖譜專家路由與 BFS 的覆蓋率。
    > 
    > 在我們的**消融實驗 (Ablation Study)** 中，對比『純向量』與『純圖譜』檢索，本系統採用的『雙層路由 + 物理回溯』混合架構，在 Context Recall 與 Faithfulness 上均取得了顯著的最優解（SOTA 表現）。
    > 
    > **最後，關於系統的未來優化方向，我們規劃了四個學術延展點**：
    > 1.  **圖拓撲感知共嵌入空間**：計畫引入 **GraphSAGE (NeurIPS 2017)** 等 GNN 演算法，將圖拓撲結構融入連續向量表徵，進一步提升路由精準度。
    > 2.  **多源聯邦本體動態對齊**：自動在查詢前對不同知識庫的關係 Label 進行動態 Schema 轉換。
    > 3.  **圖譜鏈式思考推理 (Graph-CoT)**：讓 LLM 作為 Agent 沿著圖譜關係邊主動尋路（Multi-hop Reasoning）。
    > 4.  **主動自適應檢索**：參考 Self-RAG，在 LLM 生成過程中監控 Token 置信度，低於閾值時主動觸發圖譜檢索。
    > 
    > 總結來說，本專案成功構建了一個具備學術理論支撐、工程可擴展性、且能有效抑制幻覺的神經符號 GraphRAG 系統。謝謝各位教授，請教授指導。」

## 💡 給你的臨場簡報建議 (Presentation Tips for Professors)
1.  **眼神與氣勢**：不要背稿，要像在跟教授討論一個「你已經做出來的先進系統」。多用「我們實作了...」、「我們設計了...」而非「這篇論文說...」。
2.  **別花時間定義名詞**：教授絕對知道什麼是知識圖譜、Ontology 或 Attention。千萬不要花時間解釋 `SVO` 是什麼、`T-Box` 是什麼。直接用它們來描述你的系統。
3.  **放大創新點**：把「**符號-物理源頭回溯 (Symbolic-to-Physical Source Backtracking)**」作為你最大的賣點，告訴教授你解決了「傳統圖譜缺乏 context 細節」以及「傳統向量 RAG 無法做多跳推理」的痛點。
4.  **隨時準備被切斷**：如果教授在第 5 分鐘打斷你問問題，這代表他有興趣。這時可以直接跳到「對標 SOTA」與「評估指標」，用數據與文獻說服他。
